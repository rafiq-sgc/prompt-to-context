Real Failure #1 — NL2SQL: Believable Wrong SQL

User asks: "Show students with unpaid invoices."

flowchart TD Q[User question] --> NAIVE[ChatGPT: no schema] NAIVE --> OUT["SELECT * FROM student
WHERE invoice_status='unpaid'"] OUT --> LOOKS[✅ Looks fluent] OUT --> FAIL1[❌ Table name wrong] OUT --> FAIL2[❌ No JOIN to invoices] OUT --> FAIL3[❌ SELECT * leaks PII] 
-- ChatGPT without your schema
SELECT name, email
FROM student
WHERE invoice_status = 'unpaid';
WHY: No schema in context → model guesses common names
WHEN: Every NL question without RAG
HOW: Fluent SQL with wrong tables

FIX PATH (your app):
1. hybrid_search() → retrieves real tables + metadata
2. construct_system_prefix() → injects rules + ACL
3. ReAct agent → sql_db_schema tool (live columns)
4. validate_and_rewrite_sql() → BLOCK/REWRITE before execute

SUCCESS: SQL uses real tables, governed columns, LIMIT enforced

Real Failure #4 — NL2SQL: Security Almost Failed

Scenario: LLM drafts SQL with restricted columns (SSN, bank details).

SQL executes → PII in browser → incident → audit failure.

WHY: Model optimizes for answering, not policy.

WHEN: SELECT lists columns user should not see.

# sql_agent.py — after agent drafts SQL:
validation = validate_and_rewrite_sql(
    drafted_sql=sql_query,
    policy=effective_policy,
    current_user=current_user,
    ...
)
if validation.decision == "BLOCK":
    yield "...cannot return sensitive information..."
    return  # SQL never executes
Turning failure into success: every blocked query becomes a proof your system works — demo this live.

Real Failure #5 — Daily Life: Cursor Agent Chaos

❌ FAILURE (HOW you see it)

Prompt: "Fix the streaming bug"

  • Agent edits 8 files
  • Renames unrelated functions
  • Tests fail
  • 3 hours to undo

WHY: No scope boundary in context.

WHEN: Vague task + Agent mode + no plan.

✅ SUCCESS (same task)

  1. @file message.py — "explain stream flow" (2 min)
  2. Plan: "hypothesis + single-file fix" (3 min)
  3. Implement one change (10 min)
  4. Run tests (5 min)

FIX: Understand → Plan → Generate → Review → Test

Daily Life Success Checklist

  • Before: Can I state the task in one sentence?
  • Before: Did I attach the minimum context (@file, schema snippet)?
  • Before: Is this the right tool (chat vs IDE vs internal app)?
  • During: Did I ask for a plan before code?
  • After: Did I verify — not just read fluently?
  • After: Would I stake my name on this output?
  • Never: Paste prod credentials, PII, or full DB into public chat
  • Never: Execute AI-generated SQL without validator + review
